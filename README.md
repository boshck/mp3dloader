# AMusic Bot

Telegram-бот для поиска и загрузки музыки из VKontakte, YouTube и SoundCloud. Проект написан на **Python 3.11** с использованием **aiogram 3**, **yt-dlp** и вспомогательных сервисов (Redis, ffmpeg).

---

## Что умеет бот

- Принимает текстовые запросы и выдаёт результаты из VK API.
- Поддерживает прямые ссылки на треки и плейлисты VK/YouTube/SoundCloud (до 50 треков из плейлиста).
- Позволяет скачивать треки с учётом ограничений по размеру (≤100 МБ) и длительности (20 сек - 60 минут).
- Управляет очередями загрузок (2 активных + 3 в ожидании на пользователя), чтобы один пользователь не мог перегрузить систему.
- Кэширует результаты поиска (Redis + in-memory fallback) и повторно использует уже загруженные файлы.
- Отправляет уведомления админу о критических событиях (например, падение Redis).

### Мониторинг VK токенов

Бот автоматически отслеживает состояние VK токенов и уведомляет администратора при их истечении.

**Как это работает:**

1. При каждом запросе к VK API бот проверяет ответ на наличие ошибки `error_code: 5` (токен истёк или отозван)
2. Когда токен истекает:
   - 🔴 Токен немедленно исключается из ротации (помечается как "мёртвый")
   - 📨 Администратору приходит уведомление в Telegram с **первыми 15 символами токена** (например: `vk1.a.ABCD...`)
   - 📝 В лог записывается критическая ошибка с деталями
   - ⚠️ Бот продолжает работать с оставшимися активными токенами

**Что получает администратор:**

```
🚨 VK token expired: vk1.a.ABCD...

📋 Действия:
1. Откройте .env файл
2. Найдите и удалите токен: vk1.a.ABCD...
3. Получите новый токен от VK
4. Добавьте новый токен в VK_TOKENS
5. Перезапустите бота: systemctl restart amusic-bot

⚠️ Токен исключён из ротации до перезапуска.
Активных токенов: 4/5
```

**Важно:**
- Битые токены сохраняются только в памяти - после перезапуска бота они снова попадут в ротацию, если не удалить их из `.env`
- Если все токены истекут, бот перестанет работать с VK API
- Рекомендуется иметь минимум 3-5 токенов для надёжной работы

**Как получить новый VK токен:**
1. Перейдите на [vk.com/dev](https://vk.com/dev)
2. Создайте Standalone-приложение
3. Получите токен с правами `audio` и `offline`
4. Добавьте токен в `VK_TOKENS` (через запятую, если несколько)

## Запуск

### Локально
1. Настройте `.env`.
2. Убедитесь, что ffmpeg и yt-dlp установлены.
3. (Опционально) поднимите Redis.
4. Запустите `python main.py`.

### Продакшн
- Используйте Supervisor, systemd или Docker для автоперезапуска.
- Размещайте `assets/temp` и `logs/` на дисках с мониторингом.
- Обеспечьте доступ к Redis с минимальными задержками.
- Настройте алерты по логам (см. ниже).

## Поддерживаемые ссылки

Бот распознаёт и обрабатывает ссылки на треки и плейлисты из следующих источников:

### VK
- Одиночный трек: `vk.com/audio123_456` или `m.vk.com/audio123_456`
- Плейлист: `vk.com/music/playlist/-123_1` или `vk.com/music/playlist/-123_1_abc123`

### YouTube
- Одиночное видео: `youtube.com/watch?v=VIDEO_ID` или `youtu.be/VIDEO_ID`
- Плейлист: `youtube.com/playlist?list=PLAYLIST_ID`

### SoundCloud
- Трек: `soundcloud.com/artist/track`
- Плейлист (сет): `soundcloud.com/artist/sets/playlist`

**Примечание:** Из плейлистов извлекается до 50 валидных треков (длительность 20 сек - 60 минут, размер ≤100 МБ).

## Лимиты и управление ресурсами

- **Очереди загрузок**: 2 активные задачи + 3 в ожидании на пользователя, 30 одновременно по всему боту.
- **Плейлисты**: Извлекается до 50 первых валидных треков из каждого плейлиста (VK/YouTube/SoundCloud).
- **Временное хранилище**: `assets/temp` ограничен 10 ГБ. Очистка идёт пакетом по ~1 ГБ (LRU) при превышении лимита, а также после отправки файла.
- **Ограничения контента**: проверяются длительность трека (20 сек - 60 минут) и размер файла (≤100 МБ) до завершения скачивания.
- **Rate limiter**: Redis хранит окна для пользователей и глобальных операций. При падении Redis переключение на in-memory фиксируется логами и уведомлением для администратора.

## Наблюдение и оповещения

- `logs/bot.log` содержит runtime-информацию (уровень INFO по умолчанию). Рекомендуется настроить ротацию и доставку в централизованную систему логирования.
- `bot/utils/notifier.py` отправляет уведомления в Telegram. Используются уровни: CRITICAL, ERROR, WARNING, INFO.
- Важно отслеживать:
  - доступность Redis;
  - свободное место под `assets/temp`;
  - рост очередей загрузок;
  - частоту ошибок скачивания (особенно при работе с VK/YouTube/SoundCloud API).

## Частые проблемы

| Симптом | Возможная причина | Решение |
|---------|------------------|---------|
| Бот пишет, что Redis недоступен | Redis не запущен / неверный URL | Поднять Redis или скорректировать `REDIS_URL`. Fallback продолжит работать, но без персистентного лимитинга |
| Загрузка висит или оборвалась | Превышен лимит размера/длительности, либо источник вернул нестандартный поток | Проверить лимиты, посмотреть лог ошибки, попробовать другой источник |
| Диск стремительно заполняется | Очистка не успевает или лимит завышен | Проверить настройки `_max_storage_gb`, убедиться, что cron/health-check уведомляют о превышениях |
| Ошибка при скачивании с YouTube/SoundCloud | Проблемы с yt-dlp или блокировка | Обновить yt-dlp (`pip install --upgrade yt-dlp`), проверить доступность источника |
